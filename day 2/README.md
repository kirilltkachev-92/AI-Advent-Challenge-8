# День 2 — Формат ответа

Desktop-приложение на Kotlin (Compose) для сравнения одного и того же запроса в **4 режимах** контроля ответа через API.

## Запуск

```bash
cd "day 2"
export DEEPSEEK_API_KEY="..."   # или файл .env
./run.sh                        # GUI
./gradlew runCompare            # сравнение в терминале (для отчёта)
```

## Эксперимент

**Вопрос (одинаковый в обоих прогонах):**

> Расскажи, как приготовить борщ по классическому рецепту

### Режимы (4 radio button в UI)

| Режим | system | max_tokens | stop |
|-------|--------|------------|------|
| **Без ограничений** | `You are a helpful assistant.` | — | — |
| **Формат ответа** | 3 секции `## Кратко`, `## Шаги`, `## Итог` | — | — |
| **Ограничение длины** | baseline | `100` | — |
| **Условие завершения** | «Закончи строкой `---END---`…» | — | `["---END---"]` |

Кнопка **Сравнить** отправляет один вопрос во все 4 режима подряд.

Актуальные ответы: `./gradlew runCompare`

## Вывод

Один user-промпт с разными параметрами API даёт разную длину, структуру и способ завершения ответа. Каждый режим изолирует один вид контроля.
