# День 29. Оптимизация локальной LLM

**Оптимизация qwen2.5:14b под конкретную задачу**: превращать объявления
«🔥 День N» из чата AI Advent Challenge #8 в строгие JSON-карточки
`{day, title, theme, result, format}`. Все три рычага из задания — параметры,
квантование, prompt-шаблон — включаются по одному, и на каждом шаге меряются
качество, скорость, память и стабильность. Прогон полностью локальный:
оценка программная, против зафиксированного эталона, без облачного судьи.

По уточнениям тьютора: это **не** RAG-день («но скорее нет»), и все настройки
живут **на стороне нашего софта** — в опциях запроса `/api/chat`
(temperature, seed, num_ctx, num_predict, format), а не в GUI типа LM Studio
(«всегда надо держать в голове, что вас могут лишить доступа к чему угодно»).

## Матрица оптимизации

Каждый профиль добавляет один шаг; кейс один и тот же — 10 контрольных
объявлений (Дни 1–28, все 8 тем марафона), по 2 прогона:

| профиль | что меняется | качество (из 60) | строгий JSON | латентность | RAM |
|---|---|---|---|---|---|
| `baseline` | дефолты Ollama + наивный промпт | 40 | 0/10 | 6.2 с | 9.5 ГБ |
| `params` | + temperature 0, seed 7, num_ctx 2048, num_predict 256 | 40 | 0/10 | 5.5 с | 9.1 ГБ |
| `prompt` | + шаблон под кейс: схема, enum тем, few-shot, `format=json` | **59** | 10/10 | 3.1 с | 9.1 ГБ |
| `quant` | те же веса, но квант q3_K_M вместо q4_K_M | **60** | 10/10 | 4.2 с | **7.5 ГБ** |
| `mini` | qwen2.5:0.5b — крайняя точка «ресурсы против качества» | 51 | 10/10 | **0.4 с** | **0.5 ГБ** |

Что показал прогон (полный отчёт — [`output/report.md`](output/report.md)):

- **Параметры чинят не качество, а стабильность и скорость**: с temperature 0
  и seed 9/10 кейсов стали побайтово воспроизводимыми (0/10 на дефолтах),
  ответы короче и быстрее. Баллы не выросли: наивный промпт заставляет модель
  заворачивать JSON в ```` ```json ````-заборы и выдумывать свободные темы.
- **Главный рычаг — prompt-шаблон**: 40 → 59 баллов, вдвое быстрее,
  10/10 строгого JSON, 10/10 идентичных повторов.
- **Сюрприз квантования**: q3_K_M набрал 60/60 — даже тему Дня 25 угадал там,
  где q4_K_M промахнулся — при этом на 2 ГБ меньше RAM. Цена: ~20% медленнее
  на токен (19.5 против 25.4 ток/с).
- **0.5b**: в 16 раз быстрее и в 18 раз меньше памяти, но 51/60 —
  честная нижняя точка оси качества.

## Как устроено

```
data/cases.json (10 объявлений + эталоны) ──▶ 5 профилей × 10 кейсов × 2 прогона
профиль = модель + options(/api/chat) + system-промпт [+ format=json]
ответ ──Scoring──▶ 6 баллов: строгий JSON, day, title, theme, result, format
скорость — eval_count/eval_duration Ollama; память — /api/ps; отчёт — output/report.md
```

- **Протокол Ollama вручную** (`OllamaClient.kt`), как в Дни 26–28: `/api/chat`
  с `options` и `format`, плюс `/api/ps` (замер RAM загруженной модели)
  и выгрузка через `keep_alive: 0`, чтобы профили мерились независимо.
- **Профили** (`Profiles.kt`) — вся оптимизация как данные: опции запроса,
  наивный и инженерный промпты (схема полей, enum из 8 тем с определениями,
  few-shot на Дне 6, которого нет в контрольном наборе).
- **Оценка** (`Scoring.kt`) — детерминированная: поля 2–6 оцениваются и при
  JSON в markdown-обёртке (иначе база получила бы 0 за всё из-за одних заборов),
  но балл за «строгий JSON» при этом теряется.
- **Эталоны** (`data/cases.json`) выведены из самих объявлений и проверены
  руками; темы размечены по неделям марафона.

## Чеклист задания

- ✅ настройте параметры — temperature, seed, num_ctx (2048 вместо дефолтных
  4096 — под реальную длину промпта), num_predict; профиль `params`
- ✅ попробуйте квантование — q4_K_M против q3_K_M тех же весов 14b
  (+ 0.5b как крайняя точка); профили `quant`, `mini`
- ✅ измените prompt-шаблон под конкретный кейс — профиль `prompt`
- ✅ сравните качество до/после — 40/60 → 60/60 программной оценкой
- ✅ скорость и потребление ресурсов — ток/с и латентность из счётчиков Ollama,
  RAM из `/api/ps`, диск из `/api/tags`

## Запуск

```bash
brew install ollama && ollama serve          # если ещё не запущена
ollama pull qwen2.5:14b                      # база (~9 ГБ)
ollama pull qwen2.5:14b-instruct-q3_K_M      # квант для сравнения (~7.3 ГБ)
ollama pull qwen2.5:0.5b                     # мини-модель (~0.4 ГБ)

cd "day 29"
./run.sh      # вся матрица: 5 профилей × 10 кейсов × 2 прогона → output/report.md
./demo.sh     # демо для видео
```

Быстрая проверка: `CASE_LIMIT=2 STABILITY_RUNS=1 ./run.sh`.
