🔥 День 10. Управление контекстом: разные стратегии (без summary)

Реализуйте в агенте 3 разных стратегии управления контекстом (минимум), и переключатель между ними:

👉 Стратегия 1: Sliding Window

- храните только последние N сообщений
- всё остальное отбрасывайте

👉 Стратегия 2: Sticky Facts / Key-Value Memory

- введите отдельный блок “facts” (ключ-значение), который хранит важные данные из диалога

(например: цель, ограничения, предпочтения, решения, договорённости)

- обновляйте facts после каждого сообщения пользователя
- в запрос отправляйте: facts + последние N сообщений

👉 Стратегия 3: Branching (ветки диалога)

- сохраните checkpoint в диалоге
- создайте 2 ветки от одного места
- продолжите диалог в каждой ветке независимо
- переключайтесь между ветками

Протестируйте на одном и том же сценарии (например: “собираем ТЗ 10–15 сообщений”):

👉 прогоните сценарий на каждой стратегии
👉 сравните ответы и поведение агента

Сравните:

👉 качество ответа

В зависимости от архитектуры (OpenAI, LLaMA, Anthropic), разметка для system и user может иметь разную длину.

2. Кэширование контекста (Context Caching)
Для системного промпта у большинства провайдеров (OpenAI, Anthropic, DeepSeek) автоматически включается жесткое кэширование контекста (Prompt Caching). Провайдер "схлопывает" повторяющийся системный текст, оптимизирует его, и в итоговый подсчет для тарификации или внутреннего окна может идти совершенно другое число токенов.
Для юзер промпта кэширование работает хуже или не работает вовсе, так как система ожидает, что пользовательский ввод всегда уникален и динамичен.

3. Специфика обработки системных инструкций
Некоторые API (особенно Anthropic Claude) обрабатывают системный промпт через отдельный внутренний префикс. Текст там может токенизироваться иначе или сжиматься системными алгоритмами оптимизации окон внимания (Attention Masking) для экономии вычислительной мощности.
