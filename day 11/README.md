# День 11 — модель памяти ассистента (memory layers)

Консольный stateful-ассистент на Kotlin — **персональный фитнес-коуч** с **тремя явными
слоями памяти**. Каждый слой хранится отдельно, а отдельный роутер на каждом шаге **явно
решает, что и куда сохранить**. Специализация (а не «бот на все темы») — главный смысл:
один и тот же вопрос «составь тренировку» даёт разный ответ под цель, уровень, инвентарь
и ограничения по здоровью пользователя.

## Три слоя памяти

| Слой | Что хранит (для фитнес-коуча) | Где лежит | Время жизни |
|------|-----------|-----------|-------------|
| **Краткосрочная** | текущий диалог (последние реплики, окно N) | `memory/short-term.md` | до `/new` |
| **Рабочая** | текущая программа/цикл: упражнения, заметки, стадия | `memory/working.md` | до `/new` |
| **Долговременная** | профиль: цель, уровень, инвентарь, ограничения по здоровью, решения | `memory/long-term.md` | переживает сессии |

Каждый слой хранится в отдельном Markdown-файле, который можно открыть и прочитать глазами.

## Стартовое интервью (первый запуск)

Пока долговременная память пуста (новый пользователь или сразу после `/forget`), коуч
проводит короткую анкету — **выбор цифрой, без ввода текста**: цель тренировок, уровень
подготовки, доступный инвентарь, ограничения по здоровью. Ответы пишутся прямо в профиль
(долговременный слой) и сразу влияют на программы. Как только профиль заполнен, при следующих
запусках анкета не показывается.

## Как выбирается, что и куда сохранять

После каждой реплики пользователя отдельный вызов LLM (**роутер памяти**, `MemoryRouter`)
возвращает структурированный список записей вида `{layer, kind, key, value}`:

- `long_term` ← `profile` / `constraint` / `decision` / `knowledge` — стабильные сведения;
- `working` ← `task` / `requirement` / `note` / `stage` — данные текущей задачи;
- сам диалог **никуда не маршрутизируется** — он автоматически и есть краткосрочный слой.

В консоли каждая применённая запись печатается цветом своего слоя: видно, что именно и в какой
слой ушло (`→` принято, `⨯` отклонено для неизвестного слоя/типа).

## Как это влияет на ответы

Сборкой запроса занимается `PromptBuilder`: он подмешивает в system prompt **только включённые
слои**. Команды `/on` и `/off` позволяют наглядно проверить эффект:

- выключите **долговременную** (`/off l`) — коуч «забывает» цель/уровень/травмы и даёт обобщённую тренировку;
- выключите **краткосрочную** (`/off s`) — теряется нить текущего диалога;
- выключите **рабочую** (`/off w`) — пропадает контекст текущей программы и её стадии.

Команда `/prompt` выводит ровно то, что ушло в API на последнем шаге.

### Наглядный пример (что и проверяет challenge)

```
# анкета на старте: цель=похудение, уровень=новичок, инвентарь=только своё тело, колени беречь
> /new            # чистим диалог, профиль остаётся
> /off l          # выключаем долговременную память
> составь тренировку на сегодня   →  коуч: универсальная программа (бег, прыжки, выпады)
> /on l           # включаем долговременную обратно
> составь тренировку на сегодня   →  коуч: без прыжков/выпадов, вес тела, щадит колени
```

## Поток одного шага

1. Роутер решает, что и в какой слой сохранить → записи применяются в слои.
2. `PromptBuilder` собирает запрос из включённых слоёв.
3. Запрос уходит в LLM.
4. Реплики добавляются в краткосрочную память.
5. Каждый слой сохраняется в свой файл.

`/new` чистит краткосрочную и рабочую память, а долговременную (профиль) сохраняет —
агент продолжает узнавать пользователя. `/forget` сбрасывает все три слоя.

## Команды

```
/help            справка
/mem             показать все три слоя памяти
/prompt          промпт последнего запроса (что реально ушло в API)
/layers          какие слои сейчас подмешиваются в промпт
/on  s|w|l|all   включить слой в промпт (short / working / long)
/off s|w|l       выключить слой из промпта
/new             новая сессия (профиль сохраняется)
/forget          забыть всё, включая долговременную память
/exit | /quit    выход
```

## Запуск

Скопируйте `.env.example` в `.env` и впишите `DEEPSEEK_API_KEY`, затем:

```bash
cd "day 11"
./run.sh
```

## Тесты

```bash
./gradlew test
```

Тесты проверяют: окно краткосрочной памяти, раздельное хранение слоёв, маршрутизацию записей,
отклонение неизвестного слоя, влияние включённых слоёв на промпт, сохранение/восстановление с диска
и поведение `/new` (краткосрочная и рабочая чистятся, долговременная сохраняется).
