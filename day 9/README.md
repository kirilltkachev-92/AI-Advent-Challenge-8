# День 9 — сжатие истории диалога

Десктопный чат на Kotlin Compose: агент **сжимает старую историю** в summary и экономит токены, сохраняя последние N сообщений без изменений.

## Механизм

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `RECENT_MESSAGE_COUNT` | 6 | Последние N сообщений (user+assistant) уходят в API «как есть» |
| `COMPRESS_BATCH_SIZE` | 10 | Каждые N сообщений вне окна сжимаются в summary |

Алгоритм:

1. Полная история хранится для отображения в UI.
2. Summary хранится отдельно и подставляется в запрос вместо старых сообщений.
3. При превышении порога вызывается отдельный API-запрос на суммаризацию.
4. В запрос к модели попадает: `system` → `summary` (как пара user/assistant) → последние N сообщений → новый вопрос.

## Сравнение

Кнопка **«Запустить сравнение»** прогоняет сценарий из **50 ходов** через два агента параллельно:

| Фаза | Ходов | Зачем |
|------|-------|-------|
| **Факты** | 6 | Точные данные (имя, бюджет, стек…) — уйдут в summary |
| **Объём** | 34 | Общие вопросы — наращивают историю, запускают сжатие (~4 раза) |
| **Проверка** | 10 | Вопросы по фактам из начала — видно качество после сжатия |

На экране:

- итоговый summary и экономия токенов
- график роста prompt по ходам (синий vs зелёный)
- метки ⚡ в моменты сжатия
- side-by-side ответы на проверочные вопросы

## API-ключ

```env

```

```env

RECENT_MESSAGE_COUNT=6
COMPRESS_BATCH_SIZE=10
```
