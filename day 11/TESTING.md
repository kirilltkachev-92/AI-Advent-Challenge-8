# Как тестировать фитнес-коуча с моделью памяти (день 11)

Инструкция привязана к двум проверкам из задания:
**(1) какие данные попадают в каждый слой** и **(2) как это влияет на ответы ассистента**.
В конце — авто-тесты для быстрой регрессии.

---

## 0. Подготовка и запуск

```bash
cd "day 11"
cp .env.example .env          # впишите свой DEEPSEEK_API_KEY
./run.sh
```

При первом запуске (профиль пуст) коуч проводит **анкету** — отвечайте **цифрой** и Enter.
Порядок вопросов и варианты:

| Шаг | Вопрос | Варианты |
|-----|--------|----------|
| 1 | Какая цель тренировок? | 1) похудение  2) набор массы  3) выносливость  4) общий тонус |
| 2 | Уровень подготовки? | 1) новичок  2) средний  3) продвинутый |
| 3 | Где и с чем тренируетесь? | 1) зал  2) дома с инвентарём  3) только вес тела |
| 4 | Ограничения по здоровью? | 1) нет  2) спина  3) колени  4) плечи |

Для всех примеров ниже используйте профиль **`1 1 3 3`** = похудение · новичок · только вес тела · беречь колени.

После анкеты коуч печатает: `Записал в профиль (долговременная память): …`.

### Команды в REPL

```
/mem             показать все три слоя памяти
/prompt          показать промпт последнего запроса (что реально ушло в API)
/layers          какие слои сейчас подмешиваются в промпт
/on  s|w|l|all   включить слой в промпт (s=краткосрочная, w=рабочая, l=долговременная)
/off s|w|l       выключить слой из промпта
/new             новая сессия: чистит краткосрочную и рабочую, профиль сохраняет
/forget          забыть всё (после этого снова появится анкета)
/exit            выход
```

## Проверка 1. Какие данные попадают в каждый слой

После каждой реплики роутер печатает строки `→ [Слой/тип] …` — это явный выбор, что и куда
сохранить. Затем `/mem` показывает физически разделённые слои.

**Шаг 1.** Запустите и пройдите анкету `1 1 3 3`. Уже на этом шаге профиль (цель, уровень,
инвентарь, ограничения) записан в **долговременную** память.

**Шаг 2.** Отправьте сообщение со смешанной информацией:
```
начнём программу на похудение, добавь упражнение «планка», колено бережём
```
Ожидаемо (каждая строка — цветом своего слоя):
```
→ [Рабочая/task]              программа на похудение
→ [Рабочая/requirement]       планка
→ [Рабочая/stage]             planning
→ [Долговременная/constraint] беречь колено        (если ещё не в профиле)
```

**Шаг 3.** Посмотрите слои:
```
/mem
```
Убедитесь, что данные легли по назначению:
- **Долговременная** ← профиль из анкеты (цель, уровень, инвентарь, ограничения);
- **Рабочая** ← текущая программа, упражнения, стадия;
- **Краткосрочная** ← сами реплики диалога (роутер туда ничего не пишет — диалог *и есть*
  краткосрочный слой).

**Шаг 4 — физическое разделение.** В другом терминале откройте три отдельных файла:
```bash
cat "day 11/memory/long-term.md"
cat "day 11/memory/working.md"
cat "day 11/memory/short-term.md"
```

**Шаг 5 — граница слоёв.** Отправьте эфемерную реплику:
```
спасибо, понятно
```
В `/mem` она появится только в **Краткосрочной**; строк `→` не будет (роутер ничего не
маршрутизирует в долговременную/рабочую).

## Проверка 2. Как это влияет на ответы

Один и тот же вопрос даёт разные ответы в зависимости от того, какие слои подмешаны в промпт.

### 2а. Долговременная память (профиль/ограничения) — главный эффект

```
/forget                       # сбрасываем всё; коуч запустит анкету
1 1 3 3                        # похудение · новичок · только вес тела · беречь колени
/new                          # чистим диалог, профиль ОСТАЁТСЯ
/off l                        # выключаем долговременную память
составь короткую тренировку, только список упражнений
```
→ обобщённый ответ: универсальная программа, где есть **приседания/выпады/прыжки**,
и коуч может *переспросить* про ограничения здоровья (он вас «не знает»).

```
/on l                         # включаем долговременную обратно
составь короткую тренировку, только список упражнений
```
→ ответ под профиль: вес тела, для новичка, **без приседаний/выпадов/прыжков**, щадит колени.

> Между двумя вопросами обязателен `/new`: иначе персонализация удержится за счёт
> краткосрочной памяти и эффект именно долговременного слоя будет не виден.

### 2б. Краткосрочная память (нить диалога)

```
кстати, я тренируюсь по утрам
в какое время мы договорились тренироваться?   →  утром
/off s
в какое время мы договорились тренироваться?   →  не знает / переспросит
/on s
```

### 2в. Рабочая память (контекст текущей программы)

```
делаем программу на похудение, добавь упражнение «планка»
/off w
что у нас сейчас в программе?   →  коуч не видит программу/упражнения
/on w
что у нас сейчас в программе?   →  перечислит программу и «планку»
```

### Чем подтвердить, что дело именно в промпте

После любого вопроса:
```
/prompt
```
Покажет точный набор system-блоков, ушедших в API. При `/off l` блок
`[Долговременная память …]` физически отсутствует — отсюда и разница в ответе.
Строка под ответом `промпт из: [...]` дублирует, какие слои участвовали.

## Проверка сохранения и восстановления (stateful между запусками)

```
# 1-й запуск: пройдите анкету 1 1 3 3, задайте программу, затем /exit
# 2-й запуск:
./run.sh
```
Анкета НЕ появится (профиль уже есть). Введите `/mem` — все три слоя восстановлены из
`.md` файлов: профиль, текущая программа со стадией и диалог.

## Авто-тесты (быстрая регрессия, без обращения к API)

```bash
cd "day 11"
./gradlew test
```
10 тестов покрывают: окно краткосрочной памяти, раздельное хранение трёх слоёв, корректную
маршрутизацию записей по слоям, отклонение неизвестного слоя, влияние включённых слоёв на
промпт, сохранение/восстановление из `.md` и поведение `/new` (краткосрочная и рабочая
чистятся, долговременная сохраняется).
