# Схема данных: индекс документации

Индекс RAG хранится в `day 31/output/index-docs.json` — формат тот же,
что в Неделе 5 (день 21) и днях 28–30.

## Поля файла индекса (DocumentIndex)

| Поле | Тип | Что значит |
|---|---|---|
| `created_at` | string | момент постройки индекса (ISO-8601) |
| `embed_model` | string | модель эмбеддингов (`nomic-embed-text`) |
| `dim` | int | размерность векторов (768) |
| `strategy` | string | стратегия чанкинга: `markdown-section` |
| `corpus_sha` | string | SHA-256 корпуса — пересчёт только при смене документации |
| `chunks` | array | список чанков |

## Поля чанка (IndexedChunk)

| Поле | Тип | Что значит |
|---|---|---|
| `chunk_id` | string | стабильный id, например `day 30/README.md#3` |
| `source` | string | путь файла от корня репозитория |
| `title` | string | заголовок документа (первый `#`) |
| `section` | string | заголовок раздела, из которого взят чанк |
| `strategy` | string | `markdown-section` |
| `char_start`, `char_end` | int | границы чанка в исходном файле |
| `text` | string | текст чанка |
| `embedding` | array<float> | нормированный вектор (косинус = скалярное произведение) |

## Чанкинг

Один чанк = один раздел markdown (текст между заголовками), длинные
разделы дорезаются по абзацам до ~1500 символов. Заголовок документа и
раздела добавляются в текст чанка перед эмбеддингом, чтобы вектор знал,
о каком дне и о чём раздел.
