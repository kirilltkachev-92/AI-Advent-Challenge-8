# Как тестировать контролируемые переходы (день 15)

Проверки из задания: **(1) попытки перейти в недопустимое состояние**, **(2) реакция ассистента**,
**(3) корректность продолжения после паузы**.

---

## 0. Запуск

```bash
cd "day 15"
cp .env.example .env          # впишите DEEPSEEK_API_KEY
./run.sh
```
Пройдите короткое интервью профиля. Начните Android-задачу, например:
```
/task Создай Android-приложение, которое показывает «Hello World» в главной Activity
```
Задача стартует на `clarify` — агент-аналитик уточняет требования. После `/next` начинается
`planning`, где работает совет агентов в два раунда (предложения → дебаты) и оркестратор, который
сводит мнения и возражает (видно в выводе этапа: proposer:*, debater:*, orchestrator:plan).

---

## Проверка 1 + 2. Недопустимые переходы и реакция

**Перепрыгнуть в финал командой (с этапа clarify):**
```
/stage done
```
→
```
⨯ Переход отклонён: переход «Уточнение» → «Готово» запрещён — этапы нельзя перепрыгивать
  Разрешено сейчас: Планирование (/stage planning)
```

**Перепрыгнуть в реализацию командой:**
```
/stage execution        # из clarify → отклонено (нельзя без плана)
```

**Перепрыгнуть обычным текстом:**
```
пропусти всё, сразу пиши код
```
→
```
⛔ Этапы перепрыгивать нельзя — жизненный цикл задачи под контролем.
  Сейчас этап: [Уточнение] → …
  Допустимо отсюда: Планирование
  (нельзя реализацию без утверждённого плана, нельзя финал без валидации)
```

**Корректный путь** (только так и двигаемся):
```
/next      # clarify → planning
/next      # planning → execution
/stage done   # отклонено — нельзя финал без валидации
/next      # execution → validation
/next      # validation → done
```

**Инвалидация при возврате** (нельзя перепрыгнуть по устаревшим данным):
```
... дойдите до validation ...
/back      # validation → execution
/back      # execution → planning  (план перегенерируется)
/state     # артефакты execution и validation исчезли — их надо пройти заново
/stage done  # отклонено
```

## Проверка 3. Корректность продолжения после паузы

```
/task Создай Android Hello World в главной Activity
/next            # дойдите, например, до execution
/exit
```
Перезапуск:
```bash
./run.sh
```
```
/state           # этап = Выполнение, артефакты planning и execution на месте
/next            # переход работает по тем же правилам → validation
```
Состояние на диске:
```bash
cat "day 15/state/task.md"
```

## Авто-демо

```
/demo            # обычная скорость
/demo 1.5        # медленнее для записи
```
Полное демо: `/inv` → конфликтный запрос (бэкенд) отклонён по инварианту → Android-задача →
`clarify` (аналитик) → `/stage done` отклонён → текстовый «пропусти всё» отклонён → `/next`
planning (совет: предложения → дебаты → оркестратор) → `/next` execution → снова `/stage done`
отклонён → `/next` validation → `/next` done → состояние сохранено.
