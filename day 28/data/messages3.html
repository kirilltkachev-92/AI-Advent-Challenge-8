Существует несколько видов публичных MCP-серверов: от готовых к использованию эндпоинтов до обширных каталогов.

🚀 Готовые к использованию публичные эндпоинты

Репозиторий Public MCP Servers содержит серверы, которые уже запущены в интернете и доступны для немедленного тестирования:

· Inspector (https://kite-mcp-inspector.fly.dev/): Просмотр входящих запросов.
· Text Extractor (https://text-extractor.mcp.inevitable.fyi/mcp): Извлечение текста из контента.
· Time Server (https://time.mcp.inevitable.fyi/mcp): Функции времени.
· Everything Server (https://everything.mcp.inevitable.fyi/mcp): Многоцелевой сервер.
· Echo Server (https://echo.mcp.inevitable.fyi/mcp): Возвращает запрос для отладки.

Эти серверы предназначены для тестирования, разработки и обучения, но не для продакшена.

📚 Коллекции и каталоги серверов

· Awesome MCP Servers by punkpeye: Крупнейшая курируемая коллекция. Включает 18 бесплатных публичных серверов (GitHub, Slack, Notion и др.). Есть веб-каталог.
· MCP Server Collection by cloud-ru: Коллекция простых MCP-серверов. Большинство работают бесплатно без API-ключей: погода (Open-Meteo), веб-поиск (DuckDuckGo), информация по IP, UFC, Wikipedia, Fetch.
· TensorBlock MCP Index: Индексирует более 7,400 уникальных серверов. Доступен через веб-сайт и публичное API.
· MCP Registry: Первичный источник данных о публичных серверах.
· MCP Directory: Платформа для поиска серверов и клиентов.

⚙️ Эталонные и официальные серверы

Официальные эталонные реализации от авторов протокола (в основном для локального запуска):

· Everything: Тестовый сервер с промптами, ресурсами и инструментами.
· Fetch: Извлечение и конвертация веб-контента.
· Filesystem: Безопасные операции с файлами.
· Git: Инструменты для работы с Git-репозиториями.
· Memory: Граф знаний для постоянной памяти.
· Sequential Thinking: Рефлексивное решение проблем.
· Time: Конвертация времени и часовых поясов.

Полный список официальных интеграций доступен в репозитории MCP серверов.

💡 Как начать работу

Вы можете подключаться к публичным серверам через MCP-клиенты (например, SDK). Все серверы из первого списка принимают POST-запросы на эндпоинт /mcp. В документации есть примеры для curl и настройки в Claude Desktop.
