# День 27. Интеграция локальной LLM в приложение

**CLI-утилита «чат по истории Telegram-чата»**: разбирает HTML-экспорт чата
AI Advent Challenge #8, строит по нему локальный семантический индекс и отвечает
на вопросы о переписке через локальную LLM — с историей диалога, так что работают
уточняющие вопросы («а что участники уже успели **по нему** сделать?»). Всё крутится
на `localhost:11434`, ни одного облачного запроса.

## Как устроено

```
messages.html ──парсер──▶ 102 сообщения ──окна×8──▶ 21 фрагмент
                                                        │ /api/embed (nomic-embed-text)
вопрос ──/api/embed──▶ вектор ──cosine+keywords──▶ топ-5 фрагментов
                                                        │
system(фрагменты) + история диалога + вопрос ──/api/chat (qwen2.5:14b)──▶ ответ
```

- **Протокол Ollama написан вручную** (`OllamaClient.kt`), как и в прошлые дни —
  никаких SDK: `/api/version`, `/api/tags`, `/api/chat` (теперь с полной историей
  диалога, а не одним промптом) и новый `/api/embed` для векторов.
- **Парсер экспорта Telegram тоже вручную** (`TelegramExport.kt`) — структура
  HTML-экспорта регулярная: `message service` (смена даты), `from_name`
  (нет у склеенных `joined` — автор берётся из предыдущего), `text`.
- **Поиск локальный и гибридный** (`Retriever.kt`): сообщения режутся на
  перекрывающиеся окна по 8, каждое окно — вектор `nomic-embed-text` (137M,
  с префиксами `search_document:` / `search_query:`, как модель обучена).
  Оценка = косинусная близость + бонус за дословные слова вопроса — чистый cosine
  на русском вопросе промахивался мимо фрагмента с «День 27», гибрид находит.
- **Индекс кешируется** в `output/index.json` и пересчитывается только если
  изменился SHA-256 экспорта (первое построение ~1.5 с, дальше мгновенно).
- **История диалога честная**: в `/api/chat` уходит system с найденными
  фрагментами + последние 12 реплик user/assistant — местоимения в уточняющих
  вопросах модель разрешает сама. Команды: `/history`, `/clear`, `/exit`.
- Транскрипт сессии пишется в [`output/session.md`](output/session.md).

## Пример сессии

```
✓ Ollama 0.31.1 на http://localhost:11434 — чат: qwen2.5:14b, эмбеддинги: nomic-embed-text
✓ Экспорт: data-messages.html — 102 сообщений, 19 участников, дни: 07.07.2026, 08.07.2026
✓ Индекс: 21 фрагментов (из кеша, output/index.json)

вы> Какое задание дали в день 27?

В день 27 участники должны были интегрировать локальную модель (LLM) в реальное
приложение (CLI-утилита, телеграм-бот, веб-приложение или другое), чтобы приложение
отправляло запросы к локальной LLM и работало без облачных моделей.
Формат выполнения: видео + код.

вы> А что участники уже успели по нему сделать?

Участники активно интегрировали локальную модель в свои проекты. Например,
Anastasiya A прогнала задачи по агенту на локальной модели Gemma3:12b и отметила
положительные результаты. Также Temi4 Facts работал над созданием умного поиска
по чатам Telegram с использованием локальной LLM, преобразовав содержание одного
чата из 67000 сообщений в чанки.
```

Второй вопрос — уточняющий: «по нему» модель разрешила из истории диалога,
а фрагменты под него ретривер подобрал уже другие.

## Чеклист задания

- ✅ приложение отправляет запросы в локальную LLM — `/api/chat` и `/api/embed` на `localhost:11434`
- ✅ получает и отображает ответы — REPL с ответом, источниками (какие фрагменты чата
  использованы) и метриками (токены, ток/с, латентность из счётчиков Ollama)
- ✅ работает без облачных моделей — qwen2.5:14b + nomic-embed-text с диска,
  дальше `localhost` ни один байт не уходит

## Запуск

```bash
brew install ollama && ollama serve   # если ещё не запущена
ollama pull qwen2.5:14b               # чат-модель (~9 ГБ)
ollama pull nomic-embed-text          # эмбеддинги (~0.3 ГБ)

cd "day 27"
./run.sh      # интерактивный чат по истории
./demo.sh     # демо для видео: сервер + модели + готовые вопросы
```

Свой экспорт вместо приложенного: `EXPORT_PATH=/путь/к/messages.html ./run.sh`
(индекс пересчитается автоматически по SHA-256 файла).
