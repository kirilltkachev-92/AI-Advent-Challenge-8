# День 23. Реранкинг и фильтрация

Улучшенный RAG: после поиска добавлен **второй этап** — query rewrite, порог similarity
и LLM-реранкер с порогом отсечения. Сравнение с baseline (RAG Дня 22) на тех же
10 контрольных вопросах и с тем же слепым судьёй.

## Пайплайн

- **baseline (без фильтра/rewriting)** — День 22: вопрос → top-4 по косинусу → LLM.
- **improved** — вопрос →
  1. **query rewrite** (DeepSeek): вопрос переписывается в поисковый запрос с терминологией
     статьи (расшифровка сокращений, синонимы, конкретные сущности, которые встретятся
     в нужном разделе);
  2. широкая выборка: **top-20** кандидатов по косинусу (`CANDIDATES_K`);
  3. **порог косинуса 0.55** (`SIM_THRESHOLD`) — дешёвое отсечение заведомого мусора;
  4. **LLM-реранкер** (cross-encoder-паттерн: запрос и чанк оцениваются СОВМЕСТНО,
     балл 0–10 одним JSON-запросом на все кандидаты) — Ollama реранк-модели через API
     не отдаёт, поэтому LLM-судья — честный путь без новых зависимостей (как и обсуждали в чате);
  5. **порог реранка ≥6** (`RERANK_THRESHOLD`) → **top-4** (`FINAL_K`);
     если фильтры выели всё — fallback на лучший чанк по реранку.

Генерация и судья одинаковы в обоих режимах — сравнивается именно вклад фильтрации.
Все K и пороги настраиваются в `.env`.

## Результаты (output/evaluation.md)

| | baseline (без фильтра) | improved (rewrite + фильтр + реранк) |
|---|---|---|
| баллы судьи (макс 20) | 18 | **20** |
| ответов на 0 баллов | 1 | 0 |
| попадание в ожидаемые источники | 12 из 15 | 12 из 15 |
| чанков в контексте (сумма на 10 вопросов) | 40 | **16** |

Главные наблюдения:

- **Rewrite починил провал Дня 22 (Q2, состав обучающей выборки).** Оригинальный вопрос
  ставил раздел «2.2 Training Dataset» на 24-е место по косинусу; после переписывания
  («GPT-3 training dataset composition sampling weights Common Crawl WebText2 Books1…»)
  он вошёл в top-20 (cos 0.765), реранкер дал ему 10 и отсёк остальные 19 кандидатов.
  Improved ответил точными процентами, baseline честно сказал «в контексте нет».
- **Качество выросло при контексте в 2.5 раза меньше**: реранкер агрессивно режет
  воронку 20 → 1–3 чанка. Меньше контекста — меньше шума, дешевле генерация.
- **Порог косинуса 0.55 на этом корпусе не сработал ни разу** (все top-20 выше порога):
  корпус — одна статья, всё «в среднем похоже». Урок: на однотемном корпусе similarity
  threshold почти бесполезен, реальную работу делает cross-encoder/реранкер; порог
  косинуса становится полезен на разнородной базе из многих документов.
- topK до фильтрации (20) и после (1–4) виден в каждом ответе и в отчёте («воронка»).

## Запуск

```bash
brew install ollama && ollama serve      # если ещё не запущена
ollama pull nomic-embed-text             # модель эмбеддингов (~270 МБ)
cp .env.example .env                     # и вписать DEEPSEEK_API_KEY

./run.sh                 # индекс (если ещё нет) + сравнение на 10 вопросах
./run.sh index           # только построить индекс из docs/
./run.sh ask "Which datasets make up GPT-3's training mix?"   # оба режима бок о бок
./run.sh eval            # сравнение + отчёт output/evaluation.md
./demo.sh                # демо для видео
```
