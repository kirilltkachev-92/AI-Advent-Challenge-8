# API-описание: MCP-инструменты git

MCP-сервер дня 31 (`GitMcp.kt`) слушает JSON-RPC 2.0 на
`http://localhost:8031/mcp` (Streamable HTTP, протокол `2025-06-18`).
Поддерживает `initialize`, `notifications/initialized`, `ping`,
`tools/list`, `tools/call`. Каждый инструмент выполняет настоящую
команду `git` в корне репозитория.

## git_branch

Текущая git-ветка (минимальное требование задания).

- Параметры: нет.
- Под капотом: `git rev-parse --abbrev-ref HEAD`.
- Возвращает: имя ветки, например `main`.

## git_status

Краткое состояние рабочего дерева.

- Параметры: нет.
- Под капотом: `git status --porcelain=v1 --branch`.
- Возвращает: ветку и список изменённых/неотслеживаемых файлов.

## git_files

Список файлов, отслеживаемых git.

- Параметры: `dir` (string, необязательный) — ограничить подкаталогом,
  например `day 31`.
- Под капотом: `git ls-files -- <dir>`.
- Возвращает: до 200 путей; если файлов больше — счётчик остатка.

## git_diff

Diff незакоммиченных изменений.

- Параметры: `path` (string, необязательный) — ограничить файлом или
  каталогом.
- Под капотом: `git diff --stat` + `git diff` (обрезается до 4000 символов).
- Возвращает: сводку по файлам и сам патч.

## git_log

Последние коммиты.

- Параметры: `count` (integer, необязательный, по умолчанию 10, максимум 50).
- Под капотом: `git log --oneline -n <count>`.
- Возвращает: строки `хеш сообщение`.
