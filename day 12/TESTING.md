# Как тестировать персонализацию ассистента (день 12)

Инструкция привязана к двум проверкам из задания:
**(1) ответы для разных профилей** и **(2) что ассистент учитывает автоматически**.
В конце — авто-тесты для быстрой регрессии.

---

## 0. Подготовка и запуск

```bash
cd "day 12"
cp .env.example .env          # впишите свой DEEPSEEK_API_KEY
./run.sh
```

При первом запуске (личного профиля ещё нет) коуч проводит **интервью по персонализации** —
отвечайте **цифрой** и Enter (Enter = первый вариант). Вопросы собирают контекст и предпочтения:

| Шаг | Вопрос | Группа |
|-----|--------|--------|
| 1 | Цель тренировок? | контекст |
| 2 | Уровень подготовки? | контекст |
| 3 | Где и с чем тренируетесь? | контекст |
| 4 | Ограничение по здоровью? | ограничения |
| 5 | Насколько подробные ответы удобны? | **стиль** |
| 6 | В каком тоне общаться? | **стиль** |
| 7 | Как удобнее подавать ответ? | **формат** |

После интервью профиль (`id = you`) сохранён в `profiles/you.json` и сразу подключён к запросам.
Готовые демо-профили доступны независимо от интервью: `anna`, `igor`, `maria`.

### Команды в REPL

```
/profiles          список профилей (активный ●)
/use <id>          переключиться на профиль (anna | igor | maria | you)
/profile           показать активный профиль целиком
/persona on|off    подмешивать профиль в промпт или нет
/prompt            промпт последнего запроса (что реально ушло в API)
/mem               три слоя памяти текущего профиля
/new-profile       пройти интервью заново
/exit              выход
```

---

## Проверка 1. Ответы для разных профилей

Это главная демонстрация: **один и тот же запрос → разные ответы**. Между профилями
переключаемся командой `/use`, вопрос задаём дословно один и тот же.

```
/use igor
составь короткую тренировку на сегодня
```
→ кратко, по делу, списком, с цифрами:
`Жим лёжа 3×6 RPE 8, Подтягивания 3×6 RPE 8, Присед 3×6 RPE 8 — отдых 2-3 мин.`

```
/use anna
составь короткую тренировку на сегодня
```
→ подробно, дружелюбно, **по шагам, с эмодзи, без прыжков, бережёт колени**, без зала:
разминка → приседания у стены 12–15 → ягодичный мостик → заминка, мотивирующий тон.

```
/use maria
составь короткую тренировку на сегодня
```
→ нейтрально, сбалансированно: силовое **в поддержку бега**, беречь ахилл.

Что подтверждает разницу:
- статус-строка под каждым ответом: `профиль: Игорь (кратко, строго по делу, списком)` и т.п.;
- `/profiles` — список с пометкой активного;
- `/profile` — какие именно предпочтения сейчас в силе.

## Проверка 2. Что ассистент учитывает автоматически

Профиль подмешивается в КАЖДЫЙ запрос — это видно в самом промпте, и проверяется сравнением
«с профилем / без».

### 2а. Профиль реально уходит в API

```
/use anna
составь тренировку
/prompt
```
В выводе `/prompt` есть отдельный system-блок:
```
[Персонализация — профиль пользователя «Анна»]
Контекст: … Стиль общения: … Формат ответа: … Ограничения (соблюдать строго): …
```
Ассистент учитывает это, ничего не переспрашивая.

### 2б. Сравнение «с профилем / без» (один профиль, один вопрос)

```
/use anna
составь короткую тренировку на сегодня     →  по шагам, без прыжков, колени бережём, 😊
/persona off
составь короткую тренировку на сегодня     →  обобщённая программа: возможны прыжки/выпады,
                                              тон нейтральный, ограничения не учтены
/persona on
```
В `/prompt` при `/persona off` блок `[Персонализация — профиль …]` физически отсутствует —
отсюда и разница. Статус-строка показывает `профиль: ВЫКЛ`.

### 2в. Память персональна (бонус — stateful поверх профиля)

```
/use igor
заведи задачу: цикл на массу, 8 недель
/mem                       →  в рабочей памяти задача Игоря
/use anna
/mem                       →  у Анны своя память, задачи Игоря нет
/use igor
/mem                       →  задача Игоря на месте
```
Файлы памяти лежат по профилям: `memory/igor/…`, `memory/anna/…`.
Сам профиль (предпочтения): `profiles/<id>.json`.

---

## Проверка сохранения между запусками

```
# 1-й запуск: пройдите интервью, /exit
# 2-й запуск:
./run.sh
```
Интервью НЕ появится (профиль `you` уже есть). `/profiles` покажет все профили, активный
восстановлен из `profiles/active.txt`, память — из `memory/<id>/…`.

## Авто-тесты (быстрая регрессия, без обращения к API)

```bash
cd "day 12"
./gradlew test
```
12 тестов покрывают: round-trip профиля через диск, посев демо-профилей, **подмешивание профиля
в каждый запрос**, отключение персонализации (`/persona off`), **разные профили → разный блок
в промпте** на один и тот же запрос, **изоляцию памяти по профилям** и восстановление с диска,
неизменность профиля роутером, поведение `/new` и отказ на пустое сообщение.
