# День 34. Ассистент для работы с файлами проекта

**Ассистент, который активно работает с файлами, а не просто отвечает
текстом**: задача ставится на уровне цели («обнови README под код»), а какие
файлы открыть, где искать и что переписать — он решает сам, через
файловые MCP-инструменты.

Подопытный проект — «Инфопанель», маленькая Kotlin-утилита (погода + курс
валют) в `data/project` с намеренно устаревшим README и нарушениями правил
из `RULES.md`. Ассистент работает не с шаблоном, а с рабочей копией
`output/project`, которая восстанавливается одной командой — поэтому
**результат воспроизводим повторно** из одинакового состояния.

## Как устроено

```
цель («обнови документацию по коду»)
    │
    ▼
DeepSeek (function calling) ◀── tools/list ── файловый MCP-сервер (localhost:8034)
    │      tools/call:  list_files · read_file ·        │  песочница: пути только
    │                   search_files · write_file       │  внутри output/project
    │                                                   ▼
    │                              write_file сохраняет файл И возвращает
    │                              unified diff (LCS вручную, DiffUtil.kt)
    ▼
отчёт: что нашёл, что изменил + все diff-ы (/diff, отчёт report.md)
```

- **Инструменты** (`FileMcp.kt`): свой MCP-сервер, как в Днях 17/31/33 —
  JSON-RPC 2.0 поверх Streamable HTTP, вручную. Песочница: все пути
  разрешаются строго внутри рабочей копии, `..` наружу — ошибка.
- **Изменения сохраняются и видны как diff**: `write_file` пишет файл и
  возвращает unified diff (свой построчный LCS в `DiffUtil.kt` — git не
  используется, рабочая копия вне git). Все diff-ы сессии — команда `/diff`.
- **Ассистент сам инициирует работу с файлами** (`FileAgent.kt`): промпт
  требует начинать с `list_files`/`search_files`, читать файлы перед
  ответом и не задавать уточняющих вопросов. В задачах файлы не называются.
- **Проверка не на слово** (`Report.kt`): после каждого сценария отчёт
  программно проверяет файлы рабочей копии (README описывает CurrencyApi,
  KDoc появился, HttpFetcher не сломан и т. д.).

## Сценарии (из списка задания, минимум был 2 — сделано 3)

1. **Найти все места, где используется компонент** — «где используется
   HttpFetcher?» → `search_files` + `read_file`, список «файл:строка», без записи.
2. **Обновить документацию на основе кода** — README отстал (несуществующий
   `WeatherClient.fetchTemp`, чужой сервис, нет CurrencyApi, неверная команда
   запуска) → ассистент сверяет с `src/*` и переписывает файл.
3. **Проверить соответствие правилам** — читает `RULES.md`, находит публичные
   функции без KDoc в двух файлах и дописывает комментарии.

## Запуск

```bash
cd "day 34"
./run.sh            # консоль: свободные задачи ассистенту (нужен DEEPSEEK_API_KEY)
./run.sh reset      # восстановить рабочую копию проекта из шаблона
./run.sh report     # прогнать 3 сценария с проверками → output/report.md
./demo.sh           # скриптованная сессия для видео
```
