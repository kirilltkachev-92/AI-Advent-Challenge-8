# День 10 — стратегии управления контекстом (без summary)

Десктопный агент на Kotlin Compose с **тремя стратегиями** управления контекстом и автоматическим сравнением на одном сценарии.

## Стратегии

| # | Стратегия | Как работает |
|---|-----------|--------------|
| 1 | **Sliding Window** | В API уходят только последние N сообщений — остальное отбрасывается |
| 2 | **Sticky Facts** | После каждого сообщения LLM обновляет блок key-value фактов; в запрос: `facts + окно` |
| 3 | **Branching** | Checkpoint в диалоге → независимые ветки → переключение между ними |

Параметр `WINDOW_SIZE` (по умолчанию 6) — размер окна последних сообщений.

## Сценарий сравнения

**«Запуск подкаста „Код и кофе“»** — 20 ходов:

| Фаза | Ходов | Зачем |
|------|-------|-------|
| Старт | 6 | Название, бюджет 180 000 ₽, студия, первый гость |
| Обсуждение | 5 | Telegram, джингл, хэштеги — вытесняют ранние факты из окна |
| Промо | 3 | Habr, коллаборации, розыгрыш |
| Проверка | 6 | Вопросы по фактам из начала |

Для **Branching**: checkpoint после старта и обсуждения → ветки «Промо» и «Контент» → verify на каждой.
