# День 24. Цитаты, источники и анти-галлюцинации

RAG доработан так, что ответ ОБЯЗАТЕЛЬНО содержит: сам ответ, список источников
(chunk_id + section) и дословные цитаты из найденных чанков. Дословность и валидность
источников проверяются **кодом**, совпадение смысла ответа с цитатами — слепым судьёй.
Усиление: при релевантности ниже порога агент обязан сказать «не знаю» и попросить уточнение.

## Как устроено

- **Поиск** — пайплайн Дня 23: query rewrite → top-20 по косинусу → порог косинуса 0.55 →
  LLM-реранкер → порог ≥6 → top-4. **Fallback Дня 23 убран сознательно**: если после
  фильтров не осталось ни одного чанка, генератор даже не вызывается — агент отвечает
  «не знаю» и просит уточнить вопрос (правило анти-галлюцинаций).
- **Генерация** (`RagAgent.kt`) — строгий JSON: `{can_answer, answer, sources[], quotes[],
  clarification}`. Каждое утверждение должно опираться на цитату; цитаты — дословные
  непрерывные фрагменты чанков (купюры внутри цитаты помечаются `...`).
- **Проверки кодом** (`Validator.kt`):
  - источники непусты и ссылаются только на реально извлечённые chunk_id;
  - каждая цитата ДОСЛОВНО присутствует в тексте указанного чанка — сравнение с
    нормализацией пробелов (PDF рвёт строки) и типографики (кавычки/тире),
    цитаты с купюрами `...` проверяются пофрагментно с сохранением порядка.
- **Проверка смысла** (`Verification.kt`) — слепой судья DeepSeek: следует ли смысл
  ответа из цитат (groundedness, 0–2).
- **Режим «не знаю»** дополнительно проверяется 3 вопросами мимо базы
  (столица Франции, рецепт ризотто, GPT-4 на bar exam).

## Результаты (output/verification.md)

| проверка | результат |
|---|---|
| источники есть и валидны (chunk_id из контекста) | **10/10** |
| цитаты есть в каждом ответе | **10/10** |
| все цитаты дословны (проверка кодом) | 9/10 |
| смысл ответа совпадает с цитатами (2/2 у судьи) | 8/10 |
| отказ «не знаю» + просьба уточнить на вопросы мимо базы | **3/3** |

Наблюдения:

- **Единственный недословный «квази-цитат» — настоящая поимка галлюцинации**: модель
  процитировала «GPT-3 has special difficulty with 'common sense physics'», а в статье —
  «GPT-3 **seems to** have special difficulty…». Модель выбросила хеджирование и тем
  самым усилила утверждение — ровно то, что валидатор и должен ловить.
- Два вердикта 1/2 по смыслу — ответ содержит мелкие утверждения (из чанков, но не из
  выбранных цитат), судья честно помечает «не подкреплено цитатами».
- **«Не знаю» срабатывает на двух уровнях**: «Какая столица Франции?» отсёк ещё порог
  косинуса (20 → 0 кандидатов — вопрос вне домена корпуса; вот где similarity threshold,
  бесполезный внутри одной темы в Дне 23, становится полезен), а «GPT-4 on bar exam» —
  реранкер (кандидаты похожи по теме, но ответа не содержат: 20 → 0 после реранка).
- Из чата дня: Алексей подтвердил, что «все контрольные вопросы выше порога — отличный
  результат, если модель умеет говорить "не знаю"» — поэтому режим отказа демонстрируется
  специальными вопросами мимо базы.

## Запуск

```bash
brew install ollama && ollama serve      # если ещё не запущена
ollama pull nomic-embed-text             # модель эмбеддингов (~270 МБ)
cp .env.example .env                     # и вписать DEEPSEEK_API_KEY

./run.sh                 # индекс (если ещё нет) + полная проверка
./run.sh index           # только построить индекс из docs/
./run.sh ask "How accurately can humans detect GPT-3 news articles?"  # ответ + источники + цитаты
./run.sh ask "Какая столица Франции?"                                 # режим «не знаю»
./run.sh verify          # 10 контрольных + 3 мимо базы → output/verification.md
./demo.sh                # демо для видео
```

## Файлы

- `Main.kt` — CLI (index / ask / verify)
- `RagAgent.kt` — структурированный ответ (ответ + источники + цитаты) и правило «не знаю» (новое)
- `Validator.kt` — проверка источников и дословности цитат кодом (новое)
- `Verification.kt` — прогон 10+3 вопросов, groundedness-судья, отчёт (новое)
- из Дней 21–23: `Documents.kt`, `Chunking.kt`, `OllamaEmbedder.kt`, `IndexStore.kt`,
  `Search.kt`, `DeepSeekClient.kt`, `ControlSet.kt`, `QueryRewriter.kt`, `Reranker.kt`
