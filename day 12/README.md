# День 12 — персонализация ассистента (профиль поверх модели памяти)

Консольный персонализированный ассистент на Kotlin — **персональный фитнес-коуч**, у которого
**поверх трёх слоёв памяти** (день 11) добавлен **профиль пользователя**. Профиль задаётся явно,
**подмешивается в каждый запрос** и заставляет один и тот же вопрос звучать по-разному для разных
людей. Главная проверка дня: спросите «составь тренировку на сегодня» под разными профилями —
получите ощутимо разные ответы.

## Что нового по сравнению с днём 11

День 11 уже умел хранить профиль внутри долговременной памяти. День 12 поднимает персонализацию
в **первоклассную сущность**, отделённую от памяти:

- профиль — это **отдельный объект** `UserProfile` (а не запись в долговременном слое);
- он описывает предпочтения **тремя группами** (как в лекции) + контекст;
- профилей **несколько**, между ними можно переключаться, у каждого **своя память**;
- профиль **всегда** подмешивается в промпт отдельным блоком (можно отключить для сравнения);
- профилем управляет **пользователь** (интервью + команды), а не роутер памяти.

## Профиль пользователя

```
UserProfile
├─ context      — кто пользователь, цель, уровень, инвентарь/окружение
├─ style        — стиль:   подробность · тон · терминология · эмодзи
├─ format       — формат:  структура (по шагам / списком / текстом) · длина · давать ли цифры
└─ constraints  — ограничения (строго): что беречь / чего нельзя / что обязательно
```

Профиль хранится отдельным файлом `profiles/<id>.json`, активный помечен в `profiles/active.txt`.
Память у каждого профиля своя — `memory/<id>/{short-term,working,long-term}.md`.

## Как профиль подключается к каждому запросу

`PromptBuilder` собирает system prompt по слоям, и профиль идёт **сразу после базового промпта,
раньше памяти** — он задаёт тон всему ответу:

```
[system] базовый промпт коуча
[system] [Персонализация — профиль пользователя «…»]   ← всегда (если /persona on)
         Контекст · Стиль · Формат · Ограничения
[system] [Долговременная память — выучено в диалоге]    ← если слой включён
[system] [Рабочая память — текущая задача]              ← если слой включён
… окно краткосрочной памяти …
[user]   запрос
```

Команда `/prompt` показывает ровно то, что ушло в API, — профильный блок там виден целиком.
В статус-строке после ответа печатается, **какой профиль учтён** и его ключевые предпочтения.

## Что ассистент учитывает автоматически

Один раз заданный профиль применяется без переспрашивания: подробность и тон ответа, формат
(по шагам / списком), давать ли конкретные цифры (подходы×повторы, RPE), цель и уровень,
а ограничения по здоровью соблюдаются строго. Это и есть «персонализация»: не сам ответ хранится,
а **то, как и для кого он формируется**.

## Готовые демо-профили (для проверки «разные профили → разные ответы»)

При первом запуске подсевается три контрастных пользователя:

| id | Кто | Цель / уровень | Стиль / формат |
|----|-----|----------------|----------------|
| `anna` | новичок, дома без инвентаря | похудение / новичок | подробно, дружелюбно, по шагам, без прыжков, беречь колени |
| `igor` | продвинутый, зал | масса / продвинутый | кратко, по делу, списком, только цифры (×повторы, RPE) |
| `maria` | бегунья, улица | выносливость / средний | сбалансированно, нейтрально, силовое только в поддержку бега |

Переключение: `/use igor`. Свой профиль создаётся интервью при первом запуске (`id = you`)
или командой `/new-profile`.

### Наглядный пример (что и проверяет challenge)

```
> /use igor
> составь короткую тренировку на сегодня
коуч: Жим лёжа 3×6 RPE 8, Подтягивания 3×6 RPE 8, Присед 3×6 RPE 8 — отдых 2-3 мин.

> /use anna
> составь короткую тренировку на сегодня
коуч: Привет, Анна! 😊 … разминка по шагам, приседания у стены 12–15, без прыжков,
      колени бережём, ягодичный мостик … 💪   (подробно и мотивирующе)

> /persona off          # сравнить «без профиля»
> составь короткую тренировку на сегодня
коуч: обобщённая программа без учёта цели/уровня/ограничений
```

## Поток одного шага

1. Роутер памяти решает, что сохранить в рабочую/долговременную память (профиль он **не трогает**).
2. `PromptBuilder` собирает запрос: **профиль** + включённые слои памяти.
3. Запрос уходит в LLM.
4. Реплики добавляются в краткосрочную память активного профиля.
5. Каждый слой сохраняется в файлы профиля.

## Команды

```
/help              справка
/profiles          список профилей (активный ●)
/use <id>          переключиться на профиль (память тоже переключится)
/profile           показать активный профиль целиком
/persona on|off    подмешивать профиль в промпт или нет (демо «с профилем / без»)
/new-profile       пройти интервью и создать свой профиль заново
/mem               показать все три слоя памяти текущего профиля
/prompt            промпт последнего запроса (что реально ушло в API)
/layers            какие слои памяти подмешиваются в промпт
/on  s|w|l|all     включить слой памяти в промпт
/off s|w|l         выключить слой памяти из промпта
/new               новая сессия (профиль сохраняется)
/forget            забыть память текущего профиля (профиль остаётся)
/reset             стереть ВСЁ (все профили и их память) — как первый запуск
/exit | /quit      выход
```

## Запуск

Скопируйте `.env.example` в `.env` и впишите `DEEPSEEK_API_KEY`, затем:

```bash
cd "day 12"
./run.sh
```

## Демо для записи (команда `/demo`)

Прямо в REPL команда `/demo` сама проигрывает весь тест-сценарий с паузами и подписями на
экране — удобно записывать на видео:

```
/demo            обычная скорость
/demo 1.5        медленнее (паузы ×1.5 — удобнее для записи)
```

Сценарий: один и тот же вопрос на профилях `igor` / `anna` / `maria` (разные ответы) → тот же
вопрос с `/persona off` vs `on` (профиль реально влияет) → `/profiles`. После проигрывания
команда возвращает активный профиль и режим персонализации в исходное состояние.

## Тесты

```bash
./gradlew test
```

Тесты (12) проверяют: round-trip профиля через диск, посев демо-профилей, **подмешивание профиля
в каждый запрос**, отключение персонализации (`/persona off`), **разные профили → разный блок в
промпте** на один и тот же запрос, **изоляцию памяти по профилям** и её восстановление с диска,
неизменность профиля роутером, поведение `/new` и отказ на пустое сообщение.
