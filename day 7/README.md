# День 7 — сохранение контекста

Десктопный чат на Kotlin Compose: агент **сохраняет историю диалога** между перезапусками.

## Что нового по сравнению с днём 6

- История (`messages`) пишется в **`chat_history.json`** после каждого успешного ответа ассистента.
- При старте приложения история **загружается** из файла — диалог продолжается, как будто агент не выключался.
- **Очистить историю** — сбрасывает диалог и удаляет файл.

## Архитектура

```
UI (Main.kt)  →  ChatAgent  →  DeepSeekClient  →  DeepSeek API
                      ↓
              ChatHistoryStore  →  chat_history.json
```

## API-ключ

Переменная окружения `DEEPSEEK_API_KEY` или файл `.env` (см. `.env.example`).  

## Запуск

```bash
cd "day 7"
chmod +x run.sh
./run.sh
```

Или:

```bash
./gradlew run
```

Требуется Java 17.

## Проверка сохранения контекста

1. Запустите приложение и напишите, например: «Меня зовут Кирилл».
2. Дождитесь ответа агента и **закройте** приложение.
3. Запустите снова — в чате появятся прошлые сообщения.
4. Спросите: «Как меня зовут?» — агент должен вспомнить имя из истории.

## Команды в интерфейсе

- **Отправить** — передать сообщение агенту (с учётом всей сохранённой истории).
- **Очистить историю** — сброс диалога и удаление `chat_history.json`.
